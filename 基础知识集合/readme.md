###  ※  基本定义 Definitions

#### 1. Bit (位/比特)

A **bit** is the most basic unit of information in computing and digital communications. It can hold only one of two values: **0 or 1**.

(**Bit (位/比特)** 是计算机中最小的信息单位。它只能表示两个值之一：**0 或 1**。)

这两个值代表**二进制**系统的两个状态，常用于表示**开 / 关、真 / 假、是 / 否**等逻辑状态。

#### 2. Byt (字节)

A **byte** is a unit of digital information that most commonly consists of **8 bits**. It is the standard unit for encoding a single character in computer systems.

(**字节（byte）** 是一个数字信息的单位，通常由**8个位（bits）**组成。它是计算机中用于表示一个**字符**（如英文字母、数字、符号等）的基本单位。)

> #### 为什么一个字节是 8 位？Why does a byte consist of 8 bits?
>
> **历史背景**：在计算机发展的早期（1950～60 年代），字节的长度并不统一。不同计算机架构中，一个“byte”可能是 6 位、7 位、9 位，甚至更多。直到后期行业标准化逐步推进，才最终普遍采用 8 位结构。
>
> **为何是 8 位？** 首先，ASCII 字符集在 1963 年被提出，它用 7 位表示最多 128 个字符。为了支持校验机制或扩展控制，常常再加 1 位，因此刚好组成 8 位字节，便于字符传输和存储。其次，8 是 2 的三次幂（2³ = 8），对计算机处理器、内存对齐、总线结构等都十分友好。在硬件实现中，8 位结构便于设计且通用性强。一个 8 位单元可以表示从 0 到 255 的共 256 个不同值，既具备良好的表达能力，又易于电路实现，因此成为早期计算机体系结构的标准选择。
>
> **今天的定义**：目前绝大多数现代系统中，1 byte 就是 8 bits，已成为通用标准。然而在某些非常早期或特定用途的硬件（如嵌入式控制器）中，也曾出现过非 8 位的“字节”长度。
>
> ------
>
> #### 什么是 “一个 8 位单元”？What is an "8-bit unit"?
>
> **中文解释**：所谓的“一个 8 位单元”，指的是计算机中能够一次性处理或储存的 **8 个二进制位**（bit）所组成的 **最小数据单位**，我们称之为一个 **字节（byte）**。这 8 位可以看作是一串 0 和 1 的组合，例如：
>
> ```shell
> 01101010
> ```
>
> 这个二进制数可以表示为：
>
> - 一个整数（例如十进制 106）
> - 一个字符（如 ASCII 编码中对应字母 'j'）
> - 一段图像的灰度值或颜色通道值
> - 一个布尔位组合（如权限标志）
>
> **英文解释**：An "8-bit unit" is a group of 8 binary digits (bits), like `01101010`, which is treated as a single data block by the system. It can represent a numeric value, a character, a pixel intensity, or even a combination of flags or instructions.
>
> **技术意义**：这个“单元”是 CPU 和内存读写时的基本块。在 8 位架构的处理器中，CPU 寄存器的宽度就是 8 位，一次能处理一个字节的数据。在现代计算机中，尽管寄存器已普遍扩大到 32 位或 64 位，但“一个字节 = 8 位”这一定义仍然保留，用于表示最小寻址单位。

### ※  应用举例 Practical Examples

#### 1. 表示数据容量：

- **英文**：A plain text file containing "Hello" takes up **5 bytes**, one for each character.
- **中文**：一个包含 "Hello" 的纯文本文件占用 **5 个字节**，每个字母占用 1 字节。

#### 2. 表示图像大小：

- **英文**：An 800×600 image in 24-bit color depth consumes about 800 × 600 × 3 bytes = 1,440,000 bytes (~1.37 MB).
- **中文**：一个 800×600 像素的图像，若使用 24 位颜色深度（即每像素 3 字节），大约占用 **144 万字节（约 1.37 MB）**。

#### 3. 网络带宽单位（bit 用于速度）：

- **英文**：Internet speeds are measured in **bits per second** (bps), like 100 Mbps = 100 million bits per second.
- **中文**：网络速度以 **每秒比特数（bps）** 计量，例如 100 Mbps 表示每秒可传输 **1 亿个比特**。

> ✅ 注意：**存储容量通常以 byte 为单位（MB, GB）**，而**传输速度则以 bit 为单位（Mbps, Gbps）**。

## 1. 计算机上常用的单位

### 1.1 数据单位
计算机中数据单位是 `bit/bɪt/(比特/位)`. 在计算机内部, 数据都是以二进制的形式存储和运行的.

> 二进制数据中的一个位(bit)简写为 b, 音译为比特, 是计算机存储数据的最小单位. 一个二进制位只能表示 0 或 1 两种状态, 要表示更多的信息, 就要把多个位组合成一个整体, 一般以 8 位二进制组成一个基本单位. 

> 字节：计算机数据处理的最基本单位, 主要以字节为单位解释信息. 字节(Byte)简记为 B, 规定一个字节为 8 位, 即 1B = 8bit。每个字节由 8 个二进制位组成. 一般情况下, 一个 ASCII 码占用一个字节, 一个汉字国际码占用两个字节. 

> 字：一个字通常由一个或若干个字节组成. 字(Word)是计算机进行数据处理时, 一次存取、加工和传送的数据长度. 由于字长是计算机一次所能处理信息的实际位数, 所以, 它决定了计算机数据处理的速度, 是衡量计算机性能的一个重要指标, 字长越长, 性能越好. 

> 数据的换算关系
`1Byte = 8bit，1KB = 1024B，1MB = 1024KB，1GB = 1024MB`. 
计算机型号不同, 其字长是不同的, 常用的字长有 8、16、32 和 64 位. 一般情况下, IBM PC/XT的字长为 8 位, 80286 微机字长为 16 位, 80386/80486 微机字长为 32 位, Pentium 系列微机字长为 64 位.

## 2. `MB/S` 与 `Mbit/S` 的区别
内存的最小单位是`位 (bit)`(或称: 比特), 将 8 个位组合为一组, 称为字节(`byte`) (tip: 1byte(字节) = 8bit(比特));  每个字节都有唯一的地址, 字节地址从 0 开始, 第一位只能是 0 或 1.

计算机内存的常用单位是 
+ **千字节(KB)**: 1KB = 1024byte
+ **兆字节(MB)**: 1MB == 1024KB
+ **千兆字节(GB)**: 1GB = 1024MB
+ 大型磁盘驱动器使用 **兆兆字节(TB)**: 1TB = 1024GB

数据传输率的单位一般采用 `MB/S` 或 `Mbit/S`, 尤其在内部数据传输率上官方数据中更多的采用 `Mbit/S` 为单位. 此处有必要讲解一下两个单位二者之间的差异:
+ `MB/S`(mega byte 兆位) 的含义是**兆字节每秒**, `Mbit/S` 的含义是**兆比特每秒**, 前者是指每秒传输的字节数量, 后者是指每秒传输的比特位数.
`MB/S` 中的 `B` 字母是 `Byte` 的含义, Byte 是字节数, bit 是位数, 在计算机中每 **1Byte ＝ 8bit**, 是 1 : 8 的对应关系. 因此 **1MB/S = 8Mbit/S**.
